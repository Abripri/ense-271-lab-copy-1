## Background and Goals

### Background
A "user story map" is a way to visualize all the features a service may need. It is written in the point of view of the users and consists of three important sections: epics, steps, and stories. 

  *  Epics: Broad goals a user will want to accomplish
  
  *  Steps: The path the user will take to complete the goal
  
  *  Stories: The specific features that a user might use to complete each step

  * They are organized in the following way: <br>
    &nbsp;&nbsp;&nbsp;&nbsp;Epics <br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└── Steps <br>
    &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;└── Stories

### Goals

 * To create a user story map for a grocery delivery app.
 * To identify specific tasks that a user will need to complete in order to use the service.
 * To group the tasks that share a similar goal and display these groups in chronological order, starting with the tasks the user will       attempt first.
 * To create specific "user stories" for certain tasks. The format of which was "As a *type of user* I want to *do some goal* so that *I      accomplish some purpose*.
